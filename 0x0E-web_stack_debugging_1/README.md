## Description
This project focuses on debugging an issue related to Nginx not listening on port 80 in an Ubuntu container. The task involves identifying the problem, fixing it, and automating the solution with a Bash script. The goal is to ensure that Nginx is running and listening on port 80 of all active IPv4 IPs on the server.

## Requirements
- Ubuntu 20.04 LTS
- Bash scripting skills
- Debugging techniques
- Understanding of Nginx configuration
- Familiarity with network basics

## Tasks
### Task 0: Nginx likes port 80
- Debug the issue preventing Nginx from listening on port 80.
- Write a Bash script to automate the fix, ensuring Nginx is running and listening on port 80.

### Task 1: Make it sweet and short (Advanced)
- Use the solution from Task 0 to create a Bash script within 5 lines.
- Ensure Nginx status correctly reflects whether it's running or not.

## Repository Structure
- `0x0E-web_stack_debugging_1/`
  - `0-nginx_likes_port_80`: Bash script for Task 0.

1. Clone the repository: `git clone <repository-url>`

3. Execute the Bash script for the desired task:
